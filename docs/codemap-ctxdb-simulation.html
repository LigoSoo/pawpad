codemap · ctxdb 는 어떻게 토큰을 줄이나

파일을 전부 열지 않고 레지스트리 1줄로 위치를 찾고, 저장된 작업 기억은 명령 키워드로 필요한 것만 로드한다. 실제 프로젝트는 Project A로 익명화했다.

PawPad Toolkit · 실측 데이터 기반 인터랙티브 데모
BUG
실제 리포트(v2.41 계기): "작은 버그 하나 고치는데 context가 2~3배로 불어나고 토큰이 급증". 원인 — 에이전트가 codemap을 안 거치고 소스 전체를 뒤져서 대상을 찾았다.

아래 시뮬레이션은 같은 작업을 codemap 없이(full-scan) vs codemap 경유로 나란히 실행해, 대상을 찾기까지 읽는 토큰을 실시간으로 센다.

Act 1 · 위치 찾기

codemap — 심볼 위치 레지스트리

도메인:심볼 → 파일경로 + 시그니처 + 역할(1줄). 파일을 열지 않고 어디에 뭐가 있는지 1줄로 안다.

TARGET
작업: "퀘스트 완료 토글을 눌러도 보상이 안 들어온다" 버그 수정 → 실제 고칠 곳은 feature_controller.dart 의 FeatureCompletionController.toggle 단 하나. 문제는 에이전트가 그 위치를 모른다는 것.

Act 2 · 기억 읽기

ctxdb — 키워드 depth 컨텍스트 로더

ctxdb는 세션 재시작 전용이 아니다. 사용자의 커맨드나 첫 메시지에서 키워드를 뽑아 INDEX → L1 → L2 로 이동하고, 저장된 작업 기억 중 필요한 요약만 읽는다.

RESTART
사용자 명령: "펫 부화 UX 이어서 봐줘". ctxdb가 없으면 읽어올 L2 요약도 없다. 에이전트는 현재 상태를 알아내려고 AGENTS, 작업 메타, PRD/spec, 관련 소스를 직접 열어 상태를 재구성한다.

저장과 읽기

context-saver가 저장하고, ctxdb가 읽는다

헷갈리기 쉬운 부분은 여기다. context-saver는 현재 작업 기억을 `.ctxdb`에 남기는 저장 단계이고, ctxdb는 다음 요청에서 그 기억을 키워드로 찾아 읽는 단계다.

context-saver 호출 시점

사용자 직접 호출세션저장, save context
hook 신호Stop, PreCompact, 8턴/60% checkpoint
/checkpointlane/codemap/meta 정리 후 L2 저장
/handoffsnapshot 작성 + ctxdb 요약 보강

저장된 뒤 읽히는 방식

사용자 명령→ INDEX→ L1→ L2 요약

hook은 보통 "저장해야 한다"는 신호를 만들고, agent가 context-saver 절차로 `.ctxdb/L2`, `INDEX`, `L1` 매핑을 갱신한다. 다음 요청에서는 ctxdb가 그 저장물을 읽는다.

원리

왜 줄어드나 — 한 줄 요약

codemap

파일 read ≈ 13 tok/줄. 150줄 파일 하나가 ~2K tok. 위치를 모르면 후보 파일을 여러 개 열어야 하고, 그게 곧 context 폭증이다.

codemap은 그 "어디에 뭐가 있나"를 심볼당 1줄로 미리 적어둔다. 시그니처·역할까지 있어 파일을 안 열고도 고칠 지점을 확정 → 딱 1파일만 open.

trim-router: 큰 codemap을 _root+keywords+features/{id}로 분할해 "통째 읽기" 사고 자체를 봉쇄.

ctxdb

기억 복원 비용은 "현재 상태를 어디서 복원하느냐"에 비례. ctxdb가 없으면 요약 파일이 없어서 메타 이력, PRD/spec, 관련 소스를 읽으며 상태를 추론한다.

context-saver가 L2 요약을 저장해 두면, ctxdb는 첫 메시지 키워드로 필요한 깊이까지만 로드한다. 현재 상태가 L2에 있으면 원문 재구성을 건너뛴다.

읽기: ctxdb-navigator. 저장: context-saver. hook/checkpoint/handoff는 저장을 부르는 운영 흐름.

두 축은 관측(v2.41)과도 맞물린다: 응답의 📡 Retrieval 선언 + statusline cmap/ctx/src 실측 카운터가 "codemap을 안 거치고 full-scan했는지"를 실시간으로 드러내, 위 시나리오의 이탈을 즉시 발견한다.
