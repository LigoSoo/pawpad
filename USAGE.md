# 바이브 코딩 사용 가이드 — "이럴 땐 이 스킬"

> 평소엔 그냥 **자연어로 막 시키면 됩니다.** ("로그인 화면 만들어줘", "이 버그 고쳐줘")
> 아래 스킬은 **특정 상황에서 불러 쓰면 결과가 확 좋아지는 도우미**예요. 외울 필요 없이, 막힐 때 표에서 찾아 쓰세요.

설치는 [README.md](README.md), 자세한 작업 흐름은 [GUIDE.md](GUIDE.md) 참고.

## 1. 한눈에 — 이런 생각이 들면 이 스킬

| 이런 생각이 들 때 | 입력 | 뭘 해주나 |
|------------------|------|----------|
| "아이디어는 있는데, 빠진 기능이 나중에 튀어나올까 불안해" | `/brainstorming` | 방향 잡고(대안 2-3개+추천) → 빠진 기능 스윕(빈 상태·에러·권한 등 8축 점검) → 이번에 안 만들 것(Won't)까지 확정 — 구현 후반 기능 추가/삭제 사고 방지. 기획문서(.md)를 주면 스윕부터 바로 |
| "뭘 만들지 머릿속이 안 정리됐어" | `/clarity` | 애매한 부분을 콕 집어 질문 → 명확한 사양으로 정리 (+ 구현 경로가 갈리면 2-3 대안 제시 후 추천) |
| "화면 좀 예쁘고 일관되게 해줘" | `/design` | 색·간격·배치 규칙(디자인 토큰)을 스케일로 고정 + 반응형 + "AI 티"(간격·radius·크기 제각각) 차단 |
| "기획한 거 화면으로 미리 보고 싶어" | `/mockup` | PRD를 HTML 시안으로 — 와이어프레임/디자인, 어느 메뉴에 있는지 확인 |
| "PRD·명세·메뉴·와이어를 한 화면에서 보고 편집" | `/mockup viewer` | 통합 뷰어(spec-viewer) 4탭으로 띄워 브라우저에서 편집·제자리 저장 |
| "뷰어에서 정리한 거 스팩에 반영해줘" | `/viewer-apply` | 뷰어가 저장한 `src/viewer/*.json`을 스팩 문서에 동기 (남은 항목→스팩, 삭제→제거) |
| "AI가 시키지도 않은 걸 막 고쳐!" | `/lean-code` | "딱 시킨 것만" 고삐 잡기 (오버엔지니어링 방지) |
| "이 기능 코드가 어디 있더라?" | (자동) codemap | 위치+역할 즉시 — 안 찾아도 알려줌 |
| "이 프로젝트 구조/관례가 어떻게 돼?" | `/codebase-map` | 아키텍처·구조·관례·관심사 7축 고수준 맵 (트리 안 뒤져도 정신모델) |
| "내 계획에 허점 없나 따져봐" | `/grill-me` | 꼬치꼬치 캐물어 약점·빠진 부분 발견 (모호 용어 좁힘·코드 모순 표면화 포함) |
| "방금 얘기한 거 기획서로 정리해줘" | `/to-prd` | PRD(요구사항 문서)로 깔끔하게 |
| "이 기능/이슈 다 끝났어" | `/task-done` | 마감 절차 일괄 실행 — 완료함 이동·기록·커밋 (다음 세션이 끝난 일을 다시 제안 안 하게) |
| "대화가 길어져서 느리고 까먹어" | `/checkpoint` → 새 세션 | 작업 **도중** 상태 저장 → 새 창에서 이어서 작업 (같은 AI) |
| "다른 AI(Codex)한테 넘기고 싶어" | `/handoff codex {기능}` | 작업을 스냅샷째로 인계 (담당 이전) |
| "커밋/완료 전에 보안 문제 없나 봐줘" | `/security-check` | 하드코딩 비밀번호·API key·취약점·위험 설정 스캔 → 발견 시 완료 막고 조치 제안 |
| "다 만든 거 다른 시각으로 검토받고 싶어" | `/review` | 변경을 문서로 정리해 다른 세션/에이전트가 직접 검증 리뷰 → 받아서 수정 (codex보다 저토큰) |
| "코딩은 저렴한 모델한테 시키고 토큰 아껴" | `/code-delegate` | 설계는 상위 모델, 코딩은 고른 하위 모델 서브에이전트에 위임 → 요약만 받음 (부모 컨텍스트 절감) |
| "답변이 너무 길어, 짧게" | `/caveman` | 군더더기 빼고 압축 (켜져 있음, 끄려면 "normal mode") |
| "토큰 아끼고 싶어" | (자동) .ctxdb | 관련된 부분만 골라 읽음 |

> 입력은 슬래시 명령이에요. **Claude Code**에서는 채팅창에 `/clarity` 처럼 치면 됩니다. **Codex**는 슬래시 호출이 보장되지 않으니 "clarity 스킬 써줘"처럼 자연어로 지시하세요.
> **중요:** 이 슬래시 스킬들은 **당신이 직접 칠 때** 실행돼요. (AI가 "이거 `/clarity` 한번 쓸까요?" 제안은 할 수 있지만, 알아서 돌려주길 기대하진 마세요 — 필요할 때 직접 부르는 게 맞습니다.) 반대로 **2번 항목**은 안 쳐도 자동입니다.

## 2. 신경 안 써도 자동으로 도는 것들

이건 **당신이 부르지 않아도** 알아서 작동해요:

| 언제 | 자동 동작 |
|------|----------|
| **새 세션 시작** | 지난 작업 상태 + 코드 지도(codemap)를 자동으로 불러옴 → "어디까지 했지?" 다시 설명 안 해도 됨 |
| **매 대화마다** (Claude Code 전용) | 화면 하단 상태줄에 컨텍스트 사용량(%) 표시 → "얼마나 찼나" 한눈에 (예: `ctx 31% (62k/200k)`). Codex는 statusline 없음 |
| **매 프롬프트** | 입력 키워드로 관련 컨텍스트(L1/L2)만 자동 로드 (토큰 절약) |
| **8번 대화마다** | 작업 내용을 자동 저장 (까먹음 방지) |
| **코드 저장할 때** | 타입/문법 오류 자동 검사 (스택에 analyze 명령이 있을 때) |

→ 즉, **당신은 만들고 싶은 것만 말하면 되고**, 기록·복원·검사는 하니스가 챙깁니다.

## 3. 짧은 예시 — 자주 만나는 3가지 상황

### ① 새 기능 (막연한 아이디어 → 완성)
```
당신: "체력바 있는 미니게임 만들고 싶어"
당신: "/brainstorming"    ← 빠진 기능(게임오버·점수·재시작?)부터 펼치고 싶을 때. 방향 확실하면 생략
당신: "/clarity"          ← 머릿속이 막연할 때 당신이 직접 호출
  AI:  "웹? 모바일? 체력은 언제 깎이나요? 게임오버 화면?"
당신: "웹, 적에 닿으면 깎임, 오버되면 재시작 버튼"
당신: "/design 체력바 화면"  ← 화면 잡을 때 당신이 직접 호출
  AI:  체력바 UI(토큰·배치·반응형) 설계 → "이대로 구현해줘"
  AI:  구현 (codemap이 기존 코드 위치 자동 파악 — 이건 자동)
당신: "오늘 여기까지" → (8턴마다 자동 저장)
[다음 날] AI: (자동 복원) "어제 체력바 미니게임 만들던 중이었죠?"
```

### ② AI가 자꾸 오버할 때
```
당신: "이 버튼 색만 파란색으로"
  AI:  버튼 색 + "겸사겸사 레이아웃도..." (← 과함!)
당신: "/lean-code"
  AI:  "버튼 색만 바꿉니다."
```

### ③ 큰 프로젝트에서 특정 기능 고치기
```
당신: "관통 계산 로직 수정해줘"
  → codemap: "physics:penetration → penetration.ts, calculatePenetration(...)→결과" 자동 표시
  → AI가 그 파일만 열어봄 (전체 코드 안 뒤짐 = 빠르고 토큰 절약)
```

## 4. 풀코스 예시 — 기획부터 디자인·코딩까지 한 사이클 (전체 스킬 등장)

**목표: "운동 기록 앱에 주간 리포트 화면 추가"** — 한 기능을 처음부터 끝까지. 각 단계에 어떤 스킬이 끼는지 보세요.

### [0] 세션 시작 — `resume` + `ctxdb-navigator` (자동)
```
당신: (새 채팅 연다)
  → 자동으로 지난 작업 상태 + 관련 코드 위치 복원
  AI: "📂 지난 작업: 운동기록 목록 화면까지 완료. 오늘은 뭐 할까요?"
```
> 부를 필요 없음. 새 세션 열면 hook이 알아서 불러옴.

### [1] 빠진 기능부터 펼치기 — `/brainstorming`
```
당신: "운동 기록 앱에 리포트 기능 넣고 싶어"
당신: "/brainstorming"
  AI: 방향 후보 제시(주간 리포트 / 운동별 분석 / 목표 대비 — 추천 1개) → 체크박스로 선택
      빠진 기능 스윕: "기록이 0인 주는? 리포트 공유는? 알림으로 받아볼래요?"
      스코프 확정: Must(주간 리포트) / Should(공유) / Won't(운동별 분석 — 이번엔 안 만듦)
```
> 막연할 때 맨 앞에서 한 번. "이번엔 안 만들 것"까지 정해두면 나중에 기능이 튀어나와 갈아엎는 일이 없어요. 방향이 이미 확실하면 건너뛰고 바로 [2].

### [2] 막연한 생각 정리 — `/clarity`
```
당신: "주간 리포트 같은 거 만들고 싶어"
  AI: (/clarity) "확인할게요:
       - 무슨 지표? (총 운동시간 / 칼로리 / 횟수)
       - 막대그래프? 숫자만?
       - 이번 주 고정? 과거 주도 보기?"
당신: "총시간+칼로리, 막대그래프, 일단 이번 주만"
```

### [3] 계획 허점 따지기 — `/grill-me`
```
당신: "/grill-me"
  AI: "데이터 없는 주는 어떻게 보여줘요? 주 시작은 월요일? 일요일?
       나중에 과거 주 못 보면 답답하지 않을까요?"
당신: (답하며 빈틈 메움)
```
> 용어가 모호하거나 말한 동작이 코드와 어긋나면 `/grill-me`가 정확한 용어로 좁히고 모순을 그 자리서 짚습니다.

### [4] 사양 확정 — `/to-prd`
```
당신: "/to-prd"
  AI: 대화를 PRD로 정리 → specs/weekly-report.md 저장
      + "구현 대기(SPEC_READY)" 표시 (나중에/다른 AI가 이어받기 쉽게)
```

### [5] 화면 설계 — `/design`
```
당신: "/design 주간 리포트 화면"
  AI: 디자인 방향 선언(+사유) → 색·간격(디자인 토큰) + 컴포넌트 배치(헤더→그래프→요약) + 반응형(모바일/태블릿)
      + 일관성 체크(간격·크기·radius 전부 스케일 값만 — "AI 만든 티" 나는 제각각 값 차단)
      + 로딩/빈/에러 상태까지 빠짐없이
```

### [6] 구현 — `codemap` + `/lean-code`
```
당신: "이대로 구현해줘"
  AI: (codemap) 기존 '차트 위젯'·'운동 데이터' 위치 즉시 파악 → 재사용 (안 새로 만듦)
      (lean-code 원칙) 주간 리포트 기능만, 기존 코드 안 건드림
      새로 만든 심볼은 codemap에 등록 (다음에 또 빨리 찾게)
당신: (AI가 또 오버하면) "/lean-code"
```

### [7] 소통 압축 — `/caveman` (기본 켜짐)
```
  답변이 짧고 핵심만 나옴 → 토큰 절약 + 빠름
  길게 풀어서 보고 싶으면: "normal mode"
```

### [8] 중간 저장 — `/checkpoint` (+ `context-saver` 자동)
```
당신: "오늘 여기까지 할게"
  → 8턴마다 자동 + /checkpoint 시: context-saver가
     작업내용을 .ctxdb/L2에 저장 + 새 키워드(예: '리포트') 인덱스에 자동 추가
```

### [9] 다른 AI로 인계 — `/handoff` (필요할 때만)
```
당신: "테스트 작성은 Codex한테 시킬래"
당신: "/handoff codex weekly-report 테스트 작성 남음"
  → 현재 상태를 스냅샷으로 저장
  → Codex가 새 세션에서 그대로 이어받아 작업 (인계 누락 없음)
```

### [10] 완료 — `/task-done` → 다음 세션
```
당신: "주간 리포트 끝났어" (또는 /task-done)
  AI: 마감 절차 일괄 — 완료함 이동 + 이력 기록 + (git이면) commit
  다음에 새 채팅 열면 → resume가 "주간 리포트 완료됨" 복원
  → 끊김 없이 다음 기능으로
```

### 한 줄 요약 (스킬 흐름)
```
세션시작(resume/ctxdb) → /brainstorming(막연하면 — 발산·빠진기능·스코프) → /clarity → /grill-me
  → /to-prd → /design → /mockup(화면 시각화) → 구현(codemap+lean-code, caveman) → /security-check(커밋/완료 전) → /review(고위험 교차검증)
  → 도중이면 /checkpoint(context-saver) · 필요시 /handoff → 끝나면 /task-done(마감 일괄)
```
> **전부 외울 필요 없어요.** 흐름만 감 잡고, 막히는 지점에서 해당 스킬을 부르면 됩니다.

## 5. 자주 묻는 질문

**Q. 스킬 꼭 써야 하나요?**
A. 아니요. 평소엔 그냥 자연어로 시키세요. "이런 생각이 들 때"(1번 표)가 오면 그때 부르면 됩니다.

**Q. 명령을 외워야 하나요?**
A. 아니요. 1번 표 + 4번 흐름만 가끔 보면 됩니다. 자주 쓰는 건 `/brainstorming`(기획 첫 발산), `/clarity`(정리), `/design`(UI), `/lean-code`(오버 방지), `/task-done`(마감).

**Q. 토큰(비용) 절약이 목적인가요?**
A. 주 목적은 **"끊겨도 이어서, 헷갈림 없이"**예요. 토큰 절약은 큰 프로젝트일수록 덤. 작은 프로젝트는 큰 차이 없어요.

**Q. 답변이 너무 짧아요/딱딱해요.**
A. 압축 모드(caveman)가 기본이에요. `normal mode` 라고 치면 평범하게 돌아옵니다.

**Q. 세션은 언제 끊고, 끊기 전에 뭘 해야 하나요?**
A. 세 갈래만 기억하세요. ① 작업이 **끝났으면** `/task-done` — 그 후엔 바로 새 채팅 열어도 됩니다(별도 저장 불필요). ② 작업 **도중**인데 대화가 길어졌으면(상태줄 50~60%) `/checkpoint` 후 새 채팅에서 이어가기. ③ **다른 AI**(Codex)한테 넘길 때만 `/handoff codex {기능}`. 매번 handoff는 과합니다 — 같은 AI로 계속이면 checkpoint가 싸고 충분해요.

**Q. 처음 뭐부터?**
